  • IPv4
    1. VER(4): 4,6
    2. HLEN(4): Min 5 Max 12
    3. TOS(8)-Type of service: Low Delay, High Throughput, Reliability (8 bits)
    4. Total Length(16): Head+DATA min:20 bytes max:65,535 bytes
    5. Identification(16): Unique Packet Id for identifying the group of fragments of a single IP datagram (16 bits)
    6. Flags(3): if IP Packet is too large to handle, these ‘flags’ tells if they can be fragmented or not. MSB:0
    7. Fragment Offset(13):exact position of the fragment in the original IP Packet.
    8. Time to live(8): how many routers (hops) this packet can cross. At each hop, its value is decremented by one and when the value reaches zero, the packet is discarded.
    9. Protocol(8): Name of proto to which the data is to be passed
    10. Header Checksum(16): if the packet is received error-free.
    11. Source IP address: 32 bits IP address of the sender
    12. Destination IP address: 32 bits IP address of the receiver
    13. Option: Optional information such as source route, record route. Used by the Network administrator to check whether a path is working or not.
